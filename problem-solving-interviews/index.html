Problem Solving Interviews

  • Relate the problem solving question to something you would actually see on the job, not abstract questions.
  • Brain-teasers and trick questions are not good interview questions.

Example Problem

Region names are stored in the database as a string in the format "Country,State,City", e.g. "Australia,QLD,Brisbane".

In the front-end, we would like to display the region name to the user so that the words appear in the reverse order, e.g. "Brisbane,QLD,Australia".

Implement a function for reversing the words in a region name string. We do not need to keep the original string.

Example Problem 2

A string is a sequence of characters. A string may encode an integer, e.g., “123” encodes 123.

In this problem, you are to implement methods that take a string representing an integer and return the corresponding integer, and vice versa.

You cannot use library functions like stoi in C++ and parseInt in Java.

Example Problem 3 - Data Structures

Implement a Stack using a Linked List

Skeleton Code:

public class Stack {

    public void push(Element element) {
        ...
    }

    public Element pop() {
        ...
    }
}

Stack

  • LIFO (last in, first out) data structure
  • Like a stack of plates - the last plate added to the pile is the first one removed.
  • Has two operations - push and pop
  • An element is pushed to the top of the stack and the top element is popped off.

Linked List

  • A linked list is a sequence of nodes where each node contains an object and a reference to the next node in the list.
  • The first node is called the Head and the last is called the Tail.
  • The tail's reference to next is null.
